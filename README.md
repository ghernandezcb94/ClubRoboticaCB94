¡Bienvenidos al repositorio oficial de proyectos demostrativos de nuestro Club de Robótica! Este espacio está diseñado para que los miembros aprendan desde los conceptos básicos de electrónica hasta la programación de sistemas autónomos complejos.

## 🚀 Objetivo del Repositorio
Servir como una biblioteca centralizada de código, diagramas y documentación para los proyectos desarrollados en el club, facilitando el aprendizaje colaborativo y la experimentación.

---

## 📂 Estructura del Repositorio

El repositorio está organizado por niveles de dificultad:

* **`01_Basicos/`**: Proyectos introductorios (Hola mundo, manejo de LEDs, lectura de botones).
* **`02_Sensores/`**: Uso de sensores ultrasónicos, PIR, temperatura (DHT11), infrarrojos, etc.
* **`03_Actuadores/`**: Control de servomotores, motores DC (puente H) y displays LCD.
* **`04_Proyectos_Integradores/`**: Proyectos completos (Seguidores de línea, robots esquiva-obstáculos, brazos robóticos).

---

## 🛠️ Requisitos Previos

Antes de comenzar, asegúrate de tener instalado:

1.  **Arduino IDE**: [Descárgalo aquí](https://www.arduino.cc/en/software).
2.  **Drivers**: Asegúrate de tener los drivers para tu placa (especialmente si usas clones chinos con el chip CH340).
3.  **Librerías**: Cada proyecto indicará en su propia carpeta si requiere la instalación de librerías específicas.

---

## 💻 Cómo utilizar estos proyectos

1.  **Clona el repositorio:**
    ```bash
    git clone [https://github.com/tu-usuario/tu-repositorio.git](https://github.com/tu-usuario/tu-repositorio.git)
    ```
2.  **Abre el proyecto:** Navega a la carpeta del proyecto deseado y abre el archivo `.ino` con el Arduino IDE.
3.  **Diagramas:** En cada carpeta encontrarás una carpeta `/diagramas` con imágenes o archivos de Fritzing para realizar las conexiones físicas.
4.  **Carga el código:** Conecta tu Arduino, selecciona el puerto COM correcto y haz clic en **Upload**.

## 🤝 Contribuciones

2. Sigue la estructura de carpetas mencionada arriba.
3. Asegúrate de comentar tu código adecuadamente.
4. Envía un **Pull Request** para revisión.
